Системный администратор и веб разработчик.

Я пишу статьи по системному администрированию и разработке различных систем, приложений и серверов.

Компетентность

Операционные системы
  • Windows
    Грешно не знать самую популярную систему на персональных компьютерах.
  • Windows Server
    В некоторых случаях использовал эту систему для стриминга и записи аудио-потоков для радио-эфиров.
  • Fedora
    Использую систему на пользовательских ПК и серверах. Да, на некоторых серверах. 😄
  • CentOS / Oracle Linux
    Использую эту промышленную систему исключительно на серверах.
  • Ubuntu / Debian
    Ubuntu использовал в качестве домашней системы. На Debian работает несколько серверов в организациях.
  • openSUSE
    Неплохая система с автоматическим и простым конфигуратором. Раньше использовал в качестве домашней системы.
  • Proxmox
    Использую Proxmox как платформу для развёртывания виртуальных машин.
  • OPNsense
    Работает в качестве шлюза и NAT между локальной сетью и внешним интернетом.
Приложения

Устанавливаю и настраиваю компоненты системы, например, такие, как:

  • Apache HTTPD
    Сервер для обработки веб-запросов.
  • NGINX
    Использую как proxy-сервер к Apache HTTPD, так и отдельно для отдачи контента.
  • Elasticsearch
    Работал в качестве сервера поиска. Настраивал локальную node.
  • Memcached
    Кэширующий сервер для хранения данных в RAM. Использовал его для снижения нагрузки на веб-сервера.
  • Redis
    После Memcached, перешёл на Redis.
  • MongoDB
    Использовал эту базу данных для работы с приложениями, написанными на NodeJS.
  • MySQL / MariaDB
    Стандарт для веб-приложений. Использовал как MySQL, так и MariaDB, что по сути, одно и тоже. Только в последних версиях они начали чуть отдаляться друг от друга.
  • VsFTPD
    Очень безопасный FTP-сервер.
  • Icecast
    Когда то поднимал трансляции радио-эфиров в эти ваши интернеты.
  • …и многие другие (чем я только не занимался…).
CMS / CMF
  • Drupal
    Предоставляет очень мощные возможности по управлению контентом. Люблю эту CMF.
  • Flarum
    Неплохой, развивающийся движок форума. Являюсь разработчиком и куратором русской локализации этого движка.
  • MediaWiki
    Для составления документации нет лучше инструмента, чем WIKI-система. Умею поднимать с единой системой авторизации пользователей и единым хранилищем медиа. Настраиваю WIKI-фермы.
  • WordPress
    Очень простой движок для сайтов различного назначения. Делаю небольшие модули и темы для этого движка.
  • XenForo
    Мощный движок для создания форума. От разработчиков vBulletin. Делаю небольшие модули и темы для этого движка.
Языки
  • HTML
    Использую для разработки веб-приложений.
  • CSS
    В работе предпочитаю использовать Sass / SCSS.
  • JavaScript
    Логика.
  • PHP
    Пишу дополнения, модули, модификации, расширения для веб-приложений.
  • Bash / PowerShell
    Пишу скрипты для автоматизации различных действий на Bash и PowerShell. PowerShell недавно начал изучать.
Инструменты
  • MHDD
    Диагностика диска. Тестирую поверхность блинов HDD на наличие дефектов.
  • HDAT2
    Расширенная диагностика диска. Корректирую / исправляю soft-bad сектора.
  • Windows Live CD
    Диагностика целого ПК и его комплектующих. Сделал собственный Live CD на основе Windows 11, для того, чтобы диагностировать компьютеры и их комплектующие.

Работы

Flarum

Отличный движок для контентных проектов, таких как блог или форум. Взялся за его перевод в 2015 году. В итоге, разработчики объявили мой перевод официальным и включили в свой дистрибутив.

  • Являюсь куратором русской локализации движка Flarum.
  • Открыл российское представительство Flarum - Flarus Community Center.
  • 2015/12/01
GitHub Actions Store

Хранилище со сценариями к GitHub Actions. Другие репозитории подключаются к этим сценариям, выполняя указанные инструкции для автоматизированного решения задач.

  • 2022/08/31
GitHub Package Store

Созданное на GitHub’е общее хранилище со скриптами, расширениями и дополнениями для CMF / CMS, а также место по разработке пакетов для дистрибутивов Linux. Каждый репозиторий хранилища представляет собой отдельный пакет или дополнение.

  • Пакеты для CMF / CMF, написанные на PHP, автоматически публикуются в репозитории Packagist.
  • Пакеты для дистрибутивов Linux автоматически отправляются в соответствующую систему сборки.
  • 2021/06/11
GitHub Universal AIK

Хранилище со сценариями установки для дистрибутивов Linux и MS Windows. Системы установки Linux подключаются к этому хранилищу, забирая сценарии и выполняя указанные инструкции.

  • 2022/01/19
Library Online

Во время работы часто приходится сталкиваться с задачами, требующими неординарного решения. Именно поэтому я сделал для себя библиотеку записей и назвал её скромно Library Online. В этой библиотеке я записываю найденные ответы, инструкции и советы по решению различных вопросов.

Библиотека, как и мои записи, доступна для всех и распространяется под лицензией CC BY-SA 4.0.

  • 2020/06/02
LiveStreet

Сначала мне очень не нравился LiveStreet CMS. Но потом я попробовал шаблон “Developer-Kit” от vOFFka на основе Bootstrap и с этим шаблоном LiveStreet для меня заиграл новыми красками. И так заиграл, что я выпустил для своих проектов и некоторое время поддерживал собственные модули и модификацию этого шаблона под названием “Developer-Kit MOD”, в которой я исправлял ошибки в вёрстке и обновлял компоненты.

К сожалению, LiveStreet CMS перестал обновляться, а автор этого движка отошёл от разработки.

  • 2015/10/06
